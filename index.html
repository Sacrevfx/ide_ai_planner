Metodologia PREVC

O documento segue a metodologia PREVC, que consiste em:

  • Planejamento
  • Revisão
  • Execução
  • Validação
  • Conclusão

Esta metodologia estrutura o processo de desenvolvimento, garantindo que todas as etapas sejam devidamente planejadas, revisadas, executadas, validadas e concluídas.

1. 📌 Identificação do Projeto

  • ID do Projeto: proj-localdev-ia-001
  • Título: Ambiente de Desenvolvimento Local com IA para Temas WordPress
  • Data de Criação: 2025-05-24
  • Responsável/Autor: Roger
  • Status Inicial: Rascunho

2. 🎯 Objetivo Geral

Criar um ambiente de desenvolvimento local assistido por IA, focado na criação de temas WordPress. A IA atua como um desenvolvedor sênior autônomo ou semi-assistido com suporte a modelos LLM via proxy seguro e endpoint unificado.

3. 🎯 Objetivo Principal

Estruturar e validar um ambiente com integração de modelos de linguagem rodando localmente via Ollama, integrados ao IDE por meio do llm-router e acesso remoto via Cloudflare Tunnel.

🔹 Entregáveis Esperados:

  • Ambiente funcional com suporte a temas como Elementor Pro, WooCommerce e templates premium
  • Endpoint unificado para múltiplos LLMs (API compatível com OpenAI)
  • Integração entre designer sênior e IA

4. 📌 Escopo Inicial (Scope)

  • IDE principal: Void IDE
  • Endpoint router: llm-router
  • Execução de modelos: Ollama
  • Proxy seguro: Cloudflare Tunnel
  • Diretório base: D:\wpcoderai
  • Organização modular: entre ferramentas globais e projetos isolados

5. 🚫 Restrições e Requisitos Não-Funcionais

  • Plataforma: Windows 11 (WSL opcional)
  • Hardware: NVIDIA RTX 3060 (12GB), 32GB RAM
  • Instalação: PowerShell, Git
  • Rede: Acesso somente para downloads e proxy
  • Segurança: HTTPS via túnel, sem exposição direta de portas

6. 📦 Ferramentas, Bibliotecas ou Pacotes Propostos

MODELOS:

  • ollama/llama3:latest
  • ollama/gemma3:12b-it-q4_K_M
  • ollama/dagbs/deepseek-coder-v2-lite-instruct:latest

FERRAMENTAS:

Ferramenta Detalhes Link
Void IDE IDE com IA local Void IDE
llm-router Router de múltiplos LLMs llm-router
Ollama Execução de LLM local Ollama
Cloudflare Tunnel Proxy HTTPS com domínio Cloudflare Tunnel

7. 🔐 Ameaças ou Riscos Iniciais (Threat Model)

  • Contexto excessivo: divisão entre agentes especializados
  • Falhas no proxy: monitoramento e reinicialização automática
  • Conflitos de versão: padronização via .env e config.js
  • Uso excessivo de recursos: monitoramento pelo monitor-agent

8. 🗺 Validação da Ideia — Checklist

  • ✅ IDE validada: Void IDE (backup: TRAE IDE)
  • ✅ Router validado com endpoints locais
  • ✅ Proxy HTTPS funcional (Cloudflare Tunnel)
  • ✅ Ollama testado com modelos no Windows 11
  • ✅ Modelos utilizados: Llama 2, Mistral 7B, DeepSeek-Coder, Gemma 3
  • ✅ Estrutura de diretórios e divisão de agentes definida

🚀 Guia de Instalação Passo a Passo

Este guia detalhado apresenta todos os passos necessários para configurar o ambiente de desenvolvimento local com IA para temas WordPress, desde a preparação inicial até a integração completa de todos os componentes.

🛠️ Preparação do Ambiente

1

Verificar Requisitos do Sistema

Certifique-se de que seu sistema atende aos requisitos mínimos:

  • Windows 11
  • NVIDIA RTX 3060 (12GB) ou equivalente
  • 32GB RAM
  • Pelo menos 20GB de espaço livre em disco
2

Instalar Ferramentas Básicas

Instale as ferramentas fundamentais necessárias:

  1. Instale o Git para Windows
  2. Verifique se o PowerShell está atualizado (versão 5.1 ou superior)
  3. Instale o Node.js (versão LTS recomendada)

Para verificar as versões instaladas, abra o PowerShell e execute:

git --version
$PSVersionTable.PSVersion
node --version
npm --version
3

Criar Estrutura de Diretórios

Abra o PowerShell como administrador e execute os seguintes comandos para criar a estrutura de diretórios base:

New-Item -Path "D:\wpcoderai" -ItemType Directory
cd D:\wpcoderai
mkdir environment, mcp-universal, templates, projects
cd environment
mkdir void-ide, llm-router, ollama, cloudflare
cd ..\mcp-universal
mkdir agents
cd agents
mkdir dev-agent, test-agent, content-agent, monitor-agent

🧠 Instalação e Configuração do Ollama

4

Baixar e Instalar o Ollama

Baixe e instale o Ollama para executar modelos de linguagem localmente:

cd D:\wpcoderai\environment\ollama
Invoke-WebRequest -Uri "https://ollama.com/download/OllamaSetup.exe" -OutFile "OllamaSetup.exe"
Start-Process "OllamaSetup.exe" -Wait

Siga as instruções do instalador. Após a instalação, o Ollama estará disponível como um serviço em segundo plano.

5

Baixar Modelos de IA

Baixe os modelos de IA necessários para o ambiente. Abra um novo terminal PowerShell e execute:

# Baixar Llama 3
ollama pull llama3

# Baixar Gemma 3
ollama pull gemma3:12b-it-q4_K_M

# Baixar DeepSeek Coder
ollama pull dagbs/deepseek-coder-v2-lite-instruct

Nota: O download dos modelos pode levar algum tempo, dependendo da sua conexão com a internet. Cada modelo tem vários gigabytes.

6

Testar o Funcionamento do Ollama

Verifique se o Ollama está funcionando corretamente executando um modelo:

ollama run llama3 "Olá, estou testando o ambiente de desenvolvimento com IA."

Você deve receber uma resposta do modelo, confirmando que o Ollama está funcionando corretamente.

💻 Instalação do Void IDE

7

Clonar o Repositório do Void IDE

Clone o repositório do Void IDE do GitHub:

cd D:\wpcoderai\environment
git clone https://github.com/voideditor/void.git void-ide
cd void-ide
8

Instalar Dependências e Compilar

Instale as dependências necessárias e compile o Void IDE:

npm install
npm run build

Nota: A compilação pode levar alguns minutos. Se ocorrerem erros relacionados a versões do Node.js, considere usar o nvm (Node Version Manager) para instalar a versão específica necessária.

9

Iniciar o Void IDE

Inicie o Void IDE para verificar se a instalação foi bem-sucedida:

npm run dev

O Void IDE deve ser iniciado e abrir uma janela. Você pode fechar a aplicação após confirmar que está funcionando corretamente.

🔄 Configuração do llm-router

10

Clonar o Repositório do llm-router

Clone o repositório do llm-router do GitHub:

cd D:\wpcoderai\environment
git clone https://github.com/kcolemangt/llm-router.git llm-router
cd llm-router
11

Instalar Dependências

Instale as dependências necessárias para o llm-router:

npm install
12

Configurar o llm-router

Crie os arquivos de configuração necessários:

1. Crie o arquivo config.js:

module.exports = {
  port: 3000,
  endpoints: {
    default: "http://localhost:11434",
    agents: {
      "dev-agent": "/api/dev",
      "test-agent": "/api/test",
      "content-agent": "/api/content",
      "monitor-agent": "/api/monitor"
    }
  },
  aliases: ["dev-agent", "test-agent", "content-agent", "monitor-agent"]
};

2. Crie o arquivo .env:

NODE_ENV=development
OLLAMA_HOST=http://localhost:11434
PROXY_HOST=https://seudominio.cloudflare.dev

Nota: Você atualizará o valor de PROXY_HOST após configurar o Cloudflare Tunnel.

13

Testar o llm-router

Inicie o llm-router para verificar se a configuração está correta:

npm run dev

Você deve ver mensagens indicando que o servidor está em execução na porta 3000. Pressione Ctrl+C para parar o servidor após confirmar que está funcionando.

🔒 Configuração do Cloudflare Tunnel

14

Baixar o Cloudflared

Baixe o cliente Cloudflared para criar e gerenciar túneis:

cd D:\wpcoderai\environment\cloudflare
Invoke-WebRequest -Uri "https://github.com/cloudflare/cloudflared/releases/latest/download/cloudflared-windows-amd64.exe" -OutFile "cloudflared.exe"
15

Autenticar com o Cloudflare

Autentique-se com sua conta do Cloudflare:

cd D:\wpcoderai\environment\cloudflare
.\cloudflared.exe login

Este comando abrirá uma página no navegador. Faça login na sua conta Cloudflare e autorize o acesso. Após a autenticação, um certificado será salvo automaticamente no diretório.

16

Criar um Túnel

Crie um túnel para seu ambiente de desenvolvimento:

.\cloudflared.exe tunnel create wpcoderai-tunnel

Este comando criará um túnel e gerará um arquivo de configuração com um UUID único. Anote o UUID do túnel, pois você precisará dele nas próximas etapas.

17

Configurar o Túnel

Crie um arquivo de configuração para o túnel. Substitua SEU-UUID-AQUI pelo UUID gerado na etapa anterior:

$tunnelConfig = @"
tunnel: SEU-UUID-AQUI
credentials-file: D:\wpcoderai\environment\cloudflare\.cloudflared\SEU-UUID-AQUI.json
ingress:
  - hostname: wpcoderai.yourdomain.com
    service: http://localhost:3000
  - service: http_status:404
"@

$tunnelConfig | Out-File -FilePath "D:\wpcoderai\environment\cloudflare\config.yml" -Encoding utf8

Nota: Substitua wpcoderai.yourdomain.com pelo subdomínio que você deseja usar. Este domínio deve estar registrado e gerenciado pelo Cloudflare.

18

Configurar DNS no Cloudflare

Adicione um registro CNAME no painel do Cloudflare:

  1. Faça login no Dashboard do Cloudflare
  2. Selecione seu domínio
  3. Vá para a seção DNS
  4. Adicione um registro CNAME:
    • Nome: wpcoderai (ou o subdomínio que você escolheu)
    • Conteúdo: SEU-UUID-AQUI.cfargotunnel.com
    • TTL: Auto
    • Proxy status: Proxied
  5. Salve o registro
19

Iniciar o Túnel

Inicie o túnel para testar a configuração:

cd D:\wpcoderai\environment\cloudflare
.\cloudflared.exe tunnel run wpcoderai-tunnel

Mantenha esta janela do PowerShell aberta. O túnel estará ativo enquanto o comando estiver em execução.

20

Atualizar o Arquivo .env do llm-router

Atualize o arquivo .env do llm-router com o domínio do seu túnel:

cd D:\wpcoderai\environment\llm-router
$envContent = @"
NODE_ENV=development
OLLAMA_HOST=http://localhost:11434
PROXY_HOST=https://wpcoderai.yourdomain.com
"@

$envContent | Out-File -FilePath ".env" -Encoding utf8

Nota: Substitua wpcoderai.yourdomain.com pelo subdomínio que você configurou no Cloudflare.

🔗 Integração dos Componentes

21

Configurar Agentes de IA

Configure os templates para os diferentes agentes de IA:

cd D:\wpcoderai\mcp-universal\agents\dev-agent
$devTemplate = @"
Você é um agente especializado em desenvolvimento WordPress.
Seu objetivo é auxiliar na criação de temas e plugins.
"@

$devTemplate | Out-File -FilePath "prompt_template.txt" -Encoding utf8

cd D:\wpcoderai\mcp-universal\agents\content-agent
$contentTemplate = @"
Você é um agente especializado em criação de conteúdo para WordPress.
Seu objetivo é auxiliar na criação de textos, descrições e metadados.
"@

$contentTemplate | Out-File -FilePath "content_guidelines.md" -Encoding utf8
22

Iniciar Todos os Serviços

Inicie todos os serviços em terminais separados:

1. Terminal 1 - Ollama (já deve estar em execução como serviço)

2. Terminal 2 - llm-router:

cd D:\wpcoderai\environment\llm-router
npm run dev

3. Terminal 3 - Cloudflare Tunnel:

cd D:\wpcoderai\environment\cloudflare
.\cloudflared.exe tunnel run wpcoderai-tunnel

4. Terminal 4 - Void IDE:

cd D:\wpcoderai\environment\void-ide
npm run dev
23

Configurar o Void IDE para Usar o llm-router

Configure o Void IDE para usar o endpoint do llm-router:

  1. Abra o Void IDE
  2. Vá para Configurações (ícone de engrenagem)
  3. Selecione "AI Settings"
  4. Em "API Endpoint", insira: https://wpcoderai.yourdomain.com/v1
  5. Em "API Key", insira qualquer valor (o llm-router não verifica a chave)
  6. Salve as configurações
24

Testar a Integração Completa

Teste se todos os componentes estão integrados corretamente:

  1. No Void IDE, crie um novo arquivo PHP
  2. Escreva um comentário solicitando ajuda ao assistente de IA, por exemplo: 
    // Crie uma função para registrar um tema WordPress personalizado
  3. Pressione Ctrl+Enter para ativar o assistente de IA
  4. Verifique se o assistente responde com código relevante

Se o assistente responder corretamente, a integração está funcionando!

Configuração de Inicialização Automática (Opcional)

Para facilitar o uso diário, você pode criar scripts de inicialização automática para todos os serviços. Crie um arquivo batch D:\wpcoderai\start-environment.bat com o seguinte conteúdo:

@echo off
echo Iniciando ambiente de desenvolvimento WordPress com IA...

start "llm-router" cmd /k "cd D:\wpcoderai\environment\llm-router && npm run dev"
timeout /t 5
start "Cloudflare Tunnel" cmd /k "cd D:\wpcoderai\environment\cloudflare && cloudflared.exe tunnel run wpcoderai-tunnel"
timeout /t 5
start "Void IDE" cmd /k "cd D:\wpcoderai\environment\void-ide && npm run dev"

echo Todos os serviços iniciados!
echo Acesse o Void IDE para começar a desenvolver.
